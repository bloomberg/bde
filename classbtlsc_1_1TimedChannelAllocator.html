Detailed Description

This class is a protocol (pure abstract interface) for a (synchronous) mechanism that allocates end points of communication channels supporting timed and untimed synchronous (buffered and non-buffered) read and write operations on a byte stream. A successful allocation returns the address of a channel and leaves status unmodified. If the channel address is null, then a non-negative status indicates an interruption, while a negative status indicates an inability to succeed at the present time, but does not guarantee that the allocator is invalid. An invalid allocator cannot successfully allocate a channel.

virtual Channel* btlsc::TimedChannelAllocator::allocate ( int *  status,
int  flags = 0 
) [pure virtual]

Allocate a stream-based channel. If the optionally specified flags incorporates btlsc::Flag::k_ASYNC_INTERRUPT, "asynchronous events" are permitted to interrupt this operation; by default, such events are ignored. Return the address of a channel on success, and null otherwise. On an unsuccessful allocation, load the specified status with a positive value if an asynchronous event interrupted the allocation, and with a value less than -1 (indicating an error) otherwise; status is not modified on success. A non-null channel address will remain valid until deallocated explicitly (see deallocate). An allocation that fails with a positive status is likely to succeed if retried. An allocation error does not necessarily invalidate this allocator, and so subsequent allocations may succeed. Use the isInvalid method for more information on the state of this allocator. Note that status values of 0 and -1 are not possible for this method.

Allocate a stream-based channel or interrupt after the specified absolute timeout time is reached. If the optionally specified flags incorporates btlsc::Flag::k_ASYNC_INTERRUPT, "asynchronous events" are permitted to interrupt this operation; by default, such events are ignored. Return the address of a channel on success, and null otherwise. On an unsuccessful allocation, load the specified status with 0 if timeout interrupted this operation, with a positive value if the interruption was due to an asynchronous event, and with a value less than -1 (indicating an error) otherwise; status is not modified on success. A non-null channel address will remain valid until deallocated explicitly (see deallocate). An allocation that fails with a non-negative status is likely to succeed if retried. An allocation error does not necessarily invalidate this allocator, and so subsequent allocations may succeed. Use the isInvalid method for more information on the state of this allocator. Note that if the timeout value has already passed, the allocation will still be attempted, but the attempt will not block. Also note that a status value of -1 is not possible for this method.

virtual void btlsc::TimedChannelAllocator::invalidate (  )  [pure virtual]

Place this allocator in a permanently invalid state. No subsequent allocation operations will succeed. Note that invalidating the allocator has no effect on the state of any channel managed by it nor on the listening socket, which, if established, must be closed explicitly (using the close method).
