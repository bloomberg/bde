Purpose:
Enumerate the set of channel status codes.
Classes:
btlmt::ChannelStatus namespace for channel status codes
See also:
Component btlmt_asyncchannel Component btlmt_channelpool Component btlmt_channelpoolchannel
Description:
This component provides namespace, btlmt::ChannelStatus, for the enum type listing the possible return codes for methods that operate on a channel.
Enumerators: 


  Name                      Descriptions
  ---------------------     -------------------------------------------------
  e_SUCCESS                 If the request had been a write request, then the
                            request was successfully enqueued or has been
                            successfully written synchronously.  If the
                            request had been a read request, then the request
                            was successfully enqueued.

  e_QUEUE_HIGHWATER         The write request failed because the current
                            write queue size (not including the size of the
                            message being written) is greater than the
                            configured write-queue size limit.

  e_READ_CHANNEL_DOWN       The read request failed because the read part of
                            the channel had already been closed.

  e_WRITE_CHANNEL_DOWN      The write request failed because the write part
                            of the channel had already been closed.

  e_ENQUEUE_HIGHWATER       The write request failed because the existing
                            write queue size (not including the size of the
                            message being written) is greater than the
                            enqueued queue high-water mark provided as a
                            function argument.

  e_UNKNOWN_ID              The write request failed because the channel
                            identified by an specified id does not exist.


Usage:
This section illustrates intended use of this component.
Example 1: Basic Syntax:
Suppose that we have a class MyChannel that defines the following elided interface: 

  class MyChannel {
      // This class represents a channel over which data may be sent and
      // received.

    public:
      //...
      int write(const bdlmca::Blob& blob);
          // Enqueue the specified 'blob' message to be written to this
          // channel.  Return 0 on success, and a non-zero value otherwise.
          // On error, the return value *may* equal to one of the enumerators
          // in 'btlmt::ChannelErrorWrite::Enum'.

      //...
  };


Further suppose that we have a btlb::Blob object, blob, and a MyChannel object, channel.
First, we use the write method to write blob into channel: 

  int rc = channel.write(blob);


Now, we compare the return code to the enumerators in btlmt::ChannelStatus::Enum to handle the different return conditions: 

  switch(rc) {
    case btlmt::ChannelStatus::e_SUCCESS: { // Success

        // ...

    } break;

    // We handle the different types of failures in the cases below.

    case btlmt::ChannelStatus::e_QUEUE_HIGHWATER: {

        // ...

    } break;
    case btlmt::ChannelStatus::e_WRITE_CHANNEL_DOWN: {

        // ...

    } break;
    default: { // Handle other failures.

        // ...

    }
  };
