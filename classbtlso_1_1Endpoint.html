Detailed Description

This value-semantic class characterizes a TCP, UDP, SCTP, etc. address as a hostname and port. The semantics of hostname are defined by higher-level components; it may, for instance, contain a domain name (requiring name resolution), or a dotted-decimal address. Objects can either have the default value, or syntactically valid hostname and port. The default state is signified by the port attribute being 0 and the hostname being an empty string; either one can be used as an indicator of the default value. Otherwise both hostname and port contain syntactically valid values, as defined by isValid.
